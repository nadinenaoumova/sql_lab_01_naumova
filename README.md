**Пояснение логики:**

## Задача 2. Дилеры и количество сотрудников

**Текст задания:**  
Найдите 10 дилеров, у которых меньше всего сотрудников.

**Скриншот результата (первые 10 строк):**
<img width="586" height="455" alt="image" src="https://github.com/user-attachments/assets/976da3a9-9d78-4a05-909f-26fc06ba6a8c" />


**Пояснение логики:**
- Использован `LEFT JOIN`, чтобы сохранить всех дилеров, даже тех, у кого нет сотрудников
- `COUNT(s.salesperson_id)` считает количество сотрудников для каждого дилера (если сотрудников нет, возвращается 0)
- `GROUP BY` группирует по каждому дилеру для корректного подсчета
- `ORDER BY employees` сортирует по возрастанию количества сотрудников — первыми идут дилеры с наименьшим штатом
- `LIMIT 10` показывает топ-10 дилеров, наиболее нуждающихся в расширении штата

## Задача 3. Разница цен за последний год

**Текст задания:**  
Рассчитайте разницу между `base_msrp` и `sales_amount`. Если результат отрицательный, вывести 0 (использовать `GREATEST`). 

**Скриншот результата (первые 15 строк):**

<img width="949" height="715" alt="image" src="https://github.com/user-attachments/assets/91fa2556-3cac-41f7-abcb-f97f0847b449" />


**Пояснение логики:**
- `INNER JOIN` объединяет продажи и товары по `product_id`, так как нужна информация из обеих таблиц
- `GREATEST(p.base_msrp - s.sales_amount, 0)` гарантирует, что отрицательные значения (продажа выше рекомендованной цены) будут заменены на 0
- Подзапрос динамически определяет дату "год назад" от самой поздней продажи в таблице
- `WHERE` фильтрует только продажи за последний год
- `ORDER BY price_difference DESC` сортирует по убыванию разницы, чтобы первыми видеть самые большие скидки
- Округление до двух знаков после запятой делает результат более читаемым
