# Лабораторная работа №3: Агрегации и аналитические функции

**Вариант 14**

## Цель работы
Научиться выполнять быстрый и эффективный анализ данных, используя возможности агрегирования SQL. Освоить синтаксис группировки данных, фильтрации групп и вычисления статистических показателей.

## Ход выполнения работы

### 1. Теоретическая часть
В ходе подготовки были изучены следующие концепции:
*   **Агрегатные функции:** `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`, `STDDEV`. Их назначение — вычисление одного итогового значения по набору строк.
*   **Группировка `GROUP BY`:** Разделение данных на логические группы по значениям одного или нескольких столбцов для применения агрегатных функций к каждой группе отдельно.
*   **Фильтрация групп `HAVING`:** Применяется для фильтрации результатов группировки (в отличие от `WHERE`, который фильтрует строки до группировки).
*   **Аналитические функции:** Понимание работы функций для вычисления медианы (`PERCENTILE_CONT`) и моды (`MODE`).

### 2. Практическая часть
Все задания выполнялись на дампе базы данных `bi_sql_data_student_dump.sql`.

## Итоговые выводы по работе
В ходе выполнения лабораторной работы были достигнуты следующие результаты:
1.  **Освоены базовые агрегатные функции:** Успешно применены функции COUNT для подсчета пропусков через CASE, а также SUM и MIN для финансового и регионального анализа.
2.  **Закреплено понимание `GROUP BY`:** Выполнена группировка данных по типам продуктов (выручка по категориям) и по штатам (минимальные продажи).
3.  **Понято различие `WHERE` и `HAVING`:** На практическом примере продемонстрировано, что WHERE фильтрует строки до группировки, а HAVING фильтрует уже готовые группы после агрегации (отбор штатов по минимальной сумме продажи).
4.  **Навыки анализа:** Определена структура выручки: автомобили являются основным источником дохода. Выявлены "премиальные" регионы (OR, SD, RI, WY) с высоким минимальным чеком, что может быть использовано для тестирования дорогих новинок или корректировки маркетинговой стратегии.
