##  Задание 1

**Условие:**
Ранжировать сотрудников по фамилии (`last_name`) внутри гендерных групп (`gender`).

**SQL-запрос:**

```sql
SELECT 
    salesperson_id,
    first_name,
    last_name,
    gender,
    RANK() OVER (
        PARTITION BY gender 
        ORDER BY last_name
    ) AS rank_by_last_name
FROM salespeople;
```

**Пояснение:**
Данные разбиваются по полу (`PARTITION BY gender`), затем внутри каждой группы выполняется сортировка по фамилии и присваивается ранг.

##  Задание 2

**Условие:**
Вычислить прирост продаж (sales_amount - LAG(sales_amount)) по дням. 

**SQL-запрос:**

```sql
SELECT 
    sales_transaction_date::DATE AS sale_date,
    SUM(sales_amount) AS daily_sum,
    SUM(sales_amount) - LAG(SUM(sales_amount)) OVER (ORDER BY sales_transaction_date::DATE) AS sales_growth
FROM sales
GROUP BY sales_transaction_date::DATE
ORDER BY sale_date;
```

##  Задание 3

**Условие:**
Рассчитать накопительное среднее значение цены (`base_msrp`) продуктов при сортировке по дате начала производства.

**SQL-запрос:**

```sql
SELECT 
    product_id,
    model,
    production_start_date,
    base_msrp,
    AVG(base_msrp) OVER (
        ORDER BY production_start_date
    ) AS running_avg_price
FROM products
ORDER BY production_start_date;
```
**Скриншот выполнения:**
<img width="974" height="484" alt="image" src="https://github.com/user-attachments/assets/32cd56df-434d-43eb-abc0-454596ec9bed" />

**Пояснение:**
Оконная функция `AVG()` вычисляет среднее значение цены от начала набора данных до текущей строки.

##  Вывод

В ходе выполнения лабораторной работы были изучены оконные функции SQL. Были освоены:

* функции ранжирования (`RANK`)
* функции смещения (`LAG`)
* агрегатные функции в оконном режиме (`AVG`)

Оконные функции позволяют выполнять сложные аналитические вычисления без потери исходных строк, что делает их мощным инструментом для анализа данных.
